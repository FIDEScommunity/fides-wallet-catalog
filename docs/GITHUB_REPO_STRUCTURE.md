# Contributing to the FIDES Wallet Catalog

This document describes how to add or update wallet listings in the FIDES Wallet Catalog.

**Developed and maintained by FIDES Labs BV**

## How to contribute

| Route | Best for |
|-------|----------|
| **WordPress forms** (recommended) | Most wallet providers on [fides.community](https://fides.community) — guided UI, vocabulary help, moderation |
| **GitHub Pull Request** | Power users, bulk edits, automation, or contributors without a FIDES WordPress account |

Both routes produce the same `community-catalogs/<provider>/wallet-catalog.json` files in this repository (schema **v2**). After publish or merge, GitHub Actions validates, crawls, and updates `data/aggregated.json`.

### Via WordPress forms (recommended)

On the FIDES website (requires **fides-community-tools-tiles** and this plugin):

1. **Sign in** to your WordPress account on fides.community.
2. **Add a wallet** — open the page with `[fides_wallet_submit_form]`, or **suggest changes** via `[fides_wallet_update_form]` (the pencil in the wallet modal links here when configured).
3. Fill in the form (technical fields include vocabulary help popups). Pro-only sections (media gallery, recognitions, pricing, store links, etc.) are available when your organization has a **Pro** listing (linked WordPress account on the organization).
4. Submit — your entry goes to **Tools → Catalog Submissions** for review.
5. After a maintainer **publishes**, the entry syncs to this GitHub repo via the [WP Submissions Sync](https://github.com/FIDEScommunity/fides-wallet-catalog/actions/workflows/wp-submissions-sync.yml) workflow and appears in the public catalog after crawl.

**Prerequisites:** your organization should exist in the [FIDES Organization Catalog](https://github.com/FIDEScommunity/fides-organization-catalog) (same `orgId`, e.g. `org:your-org`). Add the org first if needed.

**Canonical process docs:** [fides-community-tools-tiles — Catalog Submission Governance](https://github.com/FIDEScommunity/fides-community-tools-tiles/blob/main/docs/CATALOG-SUBMISSION-GOVERNANCE.md) (§14 CI, §15 production deploy).

### Via Pull Request (alternative)

1. **Fork** the `FIDEScommunity/fides-wallet-catalog` repository.
2. **Create a directory** under `community-catalogs/`:
   - Commercial wallets: company/product slug (e.g. `my-wallet-company`)
   - Government wallets: country name in English (e.g. `netherlands`, `france`)
   - Use lowercase, hyphenated names
3. **Add or edit** `wallet-catalog.json` conforming to schema **v2** (see below).
4. **Open a Pull Request** — CI runs `npm run validate` on all community catalogs.

Do not commit test data from local WordPress imports unless it is intended for production publish.

## Repository structure

```
fides-wallet-catalog/
├── README.md
├── community-catalogs/          # One folder per provider; wallet-catalog.json each
│   ├── provider-name/
│   │   └── wallet-catalog.json
│   └── ...
├── schemas/
│   ├── wallet-catalog.schema.json      # v2 (current)
│   └── wallet-catalog-v1.schema.json   # archive only
├── wordpress-plugin/
│   └── fides-wallet-catalog/           # Display + submit/update forms
├── data/
│   ├── aggregated.json                 # Generated by crawler (public dataset)
│   └── wallet-history-state.json       # Crawler first-seen state
└── .github/workflows/
    ├── validate.yml                    # Schema validation on PR
    ├── crawl.yml                       # Daily aggregate
    └── wp-submissions-sync.yml         # WordPress publish → GitHub import + crawl
```

## Wallet catalog schema (v2)

Each `wallet-catalog.json` must use:

```json
"$schema": "https://fides.community/schemas/wallet-catalog/v2"
```

Full reference: [schemas/README.md](../schemas/README.md) and [wallet-catalog.schema.json](../schemas/wallet-catalog.schema.json).

**v2 changes (from v1):**

- `video` → `media.videos` (array, max 3)
- `certifications` (string array) → `recognitions.certifications` (`{ title, url? }`, max 10)
- New: `media.images`, `recognitions.customerStories`, `recognitions.awardsAndRecognitions`, `deploymentModel`, `slaAvailable`, `license` (+ `licenseOther` when `other`), `pricing`

**Community vs Pro export** (when tier UI is enabled): fields such as `media`, `recognitions`, `pricing`, `website`, `features`, `appStoreLinks`, `documentation`, and `repository` are included in the public GitHub export only for **Pro** organizations (linked WordPress account). `deploymentModel`, `slaAvailable`, and `license` are published for all tiers. See [schemas/README.md](../schemas/README.md#v2-tier-notes-export).

### Minimal example (v2)

Register or reuse an entry in the [organization catalog](https://github.com/FIDEScommunity/fides-organization-catalog); use the same `orgId` here.

```json
{
  "$schema": "https://fides.community/schemas/wallet-catalog/v2",
  "orgId": "org:your-org",
  "wallets": [
    {
      "id": "your-wallet",
      "name": "Your Wallet Name",
      "type": "personal",
      "platforms": ["iOS", "Android"]
    }
  ]
}
```

### Richer example (v2, Pro-oriented fields)

```json
{
  "$schema": "https://fides.community/schemas/wallet-catalog/v2",
  "orgId": "org:your-org",
  "wallets": [

      "description": "A brief description of your wallet.",

      "license": "Apache-2.0",
      "deploymentModel": "saas",
      "slaAvailable": true,
      "repository": "https://github.com/your-org/your-wallet",
```

## Required fields

| Field | Description |
|-------|-------------|
| `$schema` | `https://fides.community/schemas/wallet-catalog/v2` |
| `orgId` | Organization catalog id (e.g. `org:animo`); name, website, logo, country come from the org catalog |
| `wallets[].id` | Unique wallet id (lowercase, alphanumeric, hyphens) |
| `wallets[].name` | Display name |
| `wallets[].type` | `personal` or `organizational` |

## Optional date fields (recommended)

- `wallets[].updatedAt` (preferred) or `wallets[].updated` (legacy alias)
- top-level `lastUpdated` (catalog fallback)

`firstSeenAt` is set by the crawler — do not set it in provider JSON.

## Validation

**Pull requests:** CI runs `npm run validate` automatically.

**Locally:**

```bash
npm install
npm run validate
```

**Single file:**

```bash
npx ajv-cli validate -s schemas/wallet-catalog.schema.json \
  -d community-catalogs/your-folder/wallet-catalog.json \
  --spec=draft2020 -c ajv-formats
```

## What happens after your contribution

| Trigger | Actions |
|---------|---------|
| **PR merged** | `validate.yml` on PR; `crawl.yml` updates `data/aggregated.json` |
| **WordPress publish** | `wp-submissions-sync.yml` imports JSON, crawls (`FIDES_WALLET_SKIP_GITHUB_CRAWL=1`), commits to `main` |

The live catalog on fides.community reads from GitHub `data/aggregated.json` (with plugin cache invalidation after sync).
